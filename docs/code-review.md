# Code Review Guide

本文件定义团队默认的代码评审关注点，供 AI 和人工 review 共同使用。

## 评审目标

评审优先关注正确性、回归风险、接口兼容性、缺失验证和不必要复杂度，而不是只看代码风格。

## 评审顺序

1. 需求是否真的被满足。
2. 是否引入功能回归或兼容性问题。
3. 是否存在明显 bug、边界遗漏或错误处理缺失。
4. 验证是否真实执行，范围是否与改动匹配。
5. 实现是否过度设计或偏离项目现有结构。

## 后端重点

- 路由路径、请求参数、响应结构是否稳定。
- 错误处理是否明确，是否避免把异常直接暴露给前端。
- 环境变量、默认值和端口配置是否一致。
- 是否出现无必要的全局副作用或耦合。

## 前端重点

- 加载态、错误态、空态是否覆盖到本次需求相关路径。
- 接口调用是否复用已有服务层，是否避免重复逻辑。
- 视图是否在桌面和窄屏下保持可读。
- 展示字段是否与后端返回结构一致。

## 不接受的交付

- 声称“已验证”但没有真实命令或检查结果。
- 为了小需求引入不必要的新依赖或新框架。
- 修改无关文件且没有合理原因。
- 明知有错误仍以“应该没问题”结案。

## AI Review 提示词

可将以下提示直接交给 AI：

```md
请按 docs/code-review.md 对当前改动做 review。

要求：
- 先给 findings，按严重级别排序。
- 每条指出文件、风险和原因。
- 优先关注 bug、回归、接口兼容性、缺失验证。
- 如果没有发现问题，明确说明，并补充剩余风险或验证缺口。
```
